# Get Started with Sindre API

This guide will walk you through making your first API call to Sindre, from authentication to processing your first meeting.

## Prerequisites

Before you start, make sure you have:
- A Supabase account and Sindre workspace access
- Node.js 18+ or your preferred HTTP client
- An audio file to test with (MP3, WAV, or M4A format)

## Step 1: Authentication

Sindre uses Supabase JWT tokens for authentication. You'll need to authenticate with Supabase first to get a token.

<Tabs>
  <Tab title="Get JWT Token">
    ```javascript
    import { createClient } from '@supabase/supabase-js';
    
    const supabase = createClient(
      'https://your-project.supabase.co',
      'your-anon-key'
    );
    
    // Sign in to get JWT token
    const { data, error } = await supabase.auth.signInWithPassword({
      email: 'your-email@example.com',
      password: 'your-password'
    });
    
    const jwtToken = data.session?.access_token;
    ```
  </Tab>
  <Tab title="Use Token in API Calls">
    ```javascript
    const BASE_URL = 'https://api.sindre.com';
    
    const headers = {
      'Authorization': `Bearer ${jwtToken}`,
      'Content-Type': 'application/json'
    };
    
    // Now you can make authenticated API calls
    const response = await fetch(`${BASE_URL}/api/meetings`, {
      method: 'GET',
      headers
    });
    ```
  </Tab>
</Tabs>

<Warning>
  Keep your JWT token secure and never expose it in client-side code. Tokens expire and need to be refreshed periodically using Supabase's refresh token mechanism.
</Warning>

## Step 2: Create Your First Meeting

Start by creating a meeting record. This establishes the context for your audio processing.

```javascript
// Create a new meeting
const createMeeting = async () => {
  const response = await fetch(`${BASE_URL}/api/meetings`, {
    method: 'POST',
    headers,
    body: JSON.stringify({
      title: 'My First API Meeting',
      description: 'Testing Sindre API integration',
      scheduled_start_time: new Date().toISOString(),
      duration: 30 // minutes
    })
  });

  const meeting = await response.json();
  console.log('Meeting created:', meeting.id);
  return meeting;
};
```

## Step 3: Upload Audio for Processing

Once you have a meeting, upload an audio file for transcription and analysis.

<Tabs>
  <Tab title="Node.js">
    ```javascript
    const fs = require('fs');
    const FormData = require('form-data');

    const uploadAudio = async (meetingId, audioFilePath) => {
      const form = new FormData();
      form.append('file', fs.createReadStream(audioFilePath));
      form.append('meetingId', meetingId);

      const response = await fetch(`${BASE_URL}/api/file-upload`, {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${jwtToken}`,
          ...form.getHeaders()
        },
        body: form
      });

      const result = await response.json();
      console.log('Audio uploaded:', result);
      return result;
    };
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST \
      -H "Authorization: Bearer your_jwt_token_here" \
      -F "file=@/path/to/your/audio.mp3" \
      -F "meetingId=your_meeting_id" \
      https://api.sindre.com/api/file-upload
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import requests

    def upload_audio(meeting_id, audio_file_path):
        url = f"{BASE_URL}/api/file-upload"
        headers = {
            'Authorization': f'Bearer {jwt_token}'
        }
        
        files = {
            'file': open(audio_file_path, 'rb'),
            'meetingId': (None, meeting_id)
        }
        
        response = requests.post(url, headers=headers, files=files)
        return response.json()
    ```
  </Tab>
</Tabs>

## Step 4: Monitor Processing Status

Audio processing happens asynchronously. You can check the status of your meeting to see when transcription and analysis are complete.

```javascript
const checkMeetingStatus = async (meetingId) => {
  const response = await fetch(`${BASE_URL}/api/meetings/${meetingId}`, {
    method: 'GET',
    headers
  });

  const meeting = await response.json();
  console.log('Processing status:', meeting.processing_status);
  return meeting;
};

// Poll for completion
const waitForProcessing = async (meetingId) => {
  let meeting;
  do {
    meeting = await checkMeetingStatus(meetingId);
    if (meeting.processing_status !== 'completed') {
      await new Promise(resolve => setTimeout(resolve, 5000)); // Wait 5 seconds
    }
  } while (meeting.processing_status !== 'completed');
  
  console.log('Processing complete!');
  return meeting;
};
```

## Step 5: Retrieve Results

Once processing is complete, you can access the transcription, extracted tasks, and insights.

## Complete Example

Here's a complete workflow that ties everything together:

```javascript
const processFirstMeeting = async () => {
  try {
    // 1. Create meeting
    const meeting = await createMeeting();
    
    // 2. Upload audio
    await uploadAudio(meeting.id, './sample-meeting.mp3');
    
    // 3. Wait for processing
    await waitForProcessing(meeting.id);
    
    // 4. Get results
    const [transcription, tasks, insights] = await Promise.all([
      getTranscription(meeting.id),
      getTasks(meeting.id),
      getInsights(meeting.id)
    ]);
    
    console.log('Success! Your meeting has been processed.');
    console.log(`- Found ${tasks.length} tasks`);
    console.log(`- Generated ${insights.length} insights`);
    
  } catch (error) {
    console.error('Error processing meeting:', error);
  }
};

// Run the example
processFirstMeeting();
```

## Common Issues

<AccordionGroup>
  <Accordion icon="warning" title="Authentication Error (401)">
    Make sure your JWT token is valid and not expired. Include it in the `Authorization` header as `Bearer your_jwt_token`. If expired, refresh the token using Supabase's refresh mechanism.
  </Accordion>
  <Accordion icon="clock" title="Processing Takes Too Long">
    Processing time depends on audio length and complexity. Typical processing takes 2-3x the audio duration.
  </Accordion>
  <Accordion icon="file" title="Unsupported File Format">
    Sindre supports MP3, WAV, M4A, and FLAC formats. Files should be under 100MB.
  </Accordion>
</AccordionGroup>
