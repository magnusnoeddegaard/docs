# Task Management API

The Task Management API provides comprehensive task lifecycle management, including AI-powered task extraction from meeting transcriptions, manual task creation, assignment, and progress tracking.

## API Endpoints Overview

<CardGroup cols={2}>
  <Card title="Core Task Operations" icon="tasks">
    - `POST /api/tasks` - Create task
    - `GET /api/tasks/:taskId` - Get task details
    - `PUT /api/tasks/:taskId` - Update task
    - `DELETE /api/tasks/:taskId` - Delete task
  </Card>
  <Card title="Task Retrieval" icon="search">
    - `GET /api/tasks/user-tasks` - Get user's tasks
    - `GET /api/tasks/meeting-tasks` - Get meeting tasks
    - `GET /api/tasks/space-tasks` - Get space tasks
    - `GET /api/tasks/bet-tasks` - Get bet tasks
    - `GET /api/tasks/insight-tasks` - Get insight tasks
  </Card>
  <Card title="Task Updates" icon="pen">
    - `PATCH /api/tasks/:taskId/status` - Update status
    - `PATCH /api/tasks/:taskId/priority` - Update priority
    - `PATCH /api/tasks/:taskId/assignee` - Update assignee
    - `PATCH /api/tasks/:taskId/bet/:betId` - Update bet
  </Card>
  <Card title="Advanced Features" icon="magic">
    - `POST /api/tasks/find-all-similar-backlog` - Find similar tasks
    - `PUT /api/tasks/merge` - Merge tasks
    - `POST /api/tasks/generate-description` - AI description
    - `GET /api/tasks/:taskId/changelog` - Task history
  </Card>
</CardGroup>

<AccordionGroup>

  </Accordion>
  <Accordion icon="play" title="in_progress">
    Task is actively being worked on
  </Accordion>
  <Accordion icon="pause" title="blocked">
    Task is blocked by external dependencies
  </Accordion>
  <Accordion icon="check-circle" title="completed">
    Task has been completed successfully
  </Accordion>
  <Accordion icon="x-circle" title="cancelled">

</AccordionGroup>

## Get User Tasks

Retrieve all tasks assigned to the authenticated user.

<CodeGroup>

```javascript GET /api/tasks/user-tasks

  method: 'GET',
  headers: {
    'Authorization': `Bearer ${SINDRE_API_KEY}`,
    'Content-Type': 'application/json'
  }

const tasks = await response.json();
```

```python
response = requests.get(
```

```bash

  -H "Authorization: Bearer YOUR_API_KEY" \
```

## Create Task

Create a new task manually. Tasks can also be automatically extracted from meeting transcriptions.

<CodeGroup>

```javascript POST /api/tasks
const task = await fetch('https://api.sindre.com/api/tasks', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${SINDRE_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Implement user authentication system',
    description: 'Set up secure login/logout functionality with JWT tokens',
    priority_id: '3',
    user_id: 'user-uuid-here',
    event_id: 123,
    is_sindre_assigned: false
  })
});

const taskData = await task.json();
```

```python
import requests

response = requests.post(
    'https://api.sindre.com/api/tasks',
    headers={
        'Authorization': f'Bearer {SINDRE_API_KEY}',
        'Content-Type': 'application/json'
    },
    json={
        'name': 'Implement user authentication system',
        'description': 'Set up secure login/logout functionality with JWT tokens',
        'priority_id': '3',
        'user_id': 'user-uuid-here',
        'event_id': 123,
        'is_sindre_assigned': False
    }
)

task = response.json()
```

```bash
curl -X POST https://api.sindre.com/api/tasks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Implement user authentication system",
    "description": "Set up secure login/logout functionality with JWT tokens",
    "priority_id": "3",
    "user_id": "user-uuid-here",
    "is_sindre_assigned": false
  }'
```

</CodeGroup>

### Request Body

<ParamField body="name" type="string" required>
  Clear, actionable task name
</ParamField>

<ParamField body="description" type="string">
  Detailed description of what needs to be done
</ParamField>

<ParamField body="status" type="string" default="pending">
  Initial task status: `pending`, `in_progress`, `blocked`, `completed`, `cancelled`
</ParamField>

<ParamField body="priority_id" type="string" default="2">
  Task priority ID: 1=low, 2=medium, 3=high, 4=critical
</ParamField>

<ParamField body="status_id" type="string" default="1">
  Task status ID: 1=todo, 2=in_progress, 3=completed, etc.
</ParamField>

<ParamField body="user_id" type="string">
  User ID of the person assigned to this task (use 'sindre' for Sindre assignment)
</ParamField>

<ParamField body="is_sindre_assigned" type="boolean" default="false">
  Whether this task is assigned to Sindre AI assistant
</ParamField>

<ParamField body="event_id" type="integer">
  ID of the meeting/event this task originated from (if applicable)
</ParamField>

<ParamField body="bot_ids" type="array">
  Array of bot IDs associated with this task
</ParamField>

<ParamField body="assigned_to_name" type="string">
  Name of the person assigned to this task
</ParamField>

<ParamField body="assigned_to_email" type="string">
  Email of the person assigned to this task
</ParamField>

<ParamField body="space_id" type="string">
  Workspace/space ID (defaults to user's default space)
</ParamField>

### Response

Returns the created task object with status `201 Created`.
